- [Maestro web aria-label / resource-id resolution](ui-patterns.md): `aria-label` on web elements maps to `resource-id` in Maestro's accessibility hierarchy — NOT to the `text` attribute. `tapOn: "aria-label-value"` (plain string) will FAIL. `tapOn: {id: "aria-label-value"}` WORKS because `id:` matches resource-id. `scrollUntilVisible` with `element: text:` only matches visible DOM text, not aria-label. Plain button text (e.g. "Group Parallels", "+ Custom" visible text) IS matched by text matchers. `pressKey: Escape` does NOT fire window-level keydown events — cannot test modal Escape closure via Maestro web. CONFIRMED BROKEN: `pressKey: Tab` throws `IllegalStateException: Keycode TAB is not supported on web` — never use Tab in Maestro web flows. `pressKey: Enter` DOES work and fires the focused element's click handler.
